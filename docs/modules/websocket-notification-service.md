# 实时推送服务

## 1. 职责

该服务负责把系统内部事件实时推送到前端，支撑盯盘、告警和操作反馈。

推送内容主要包括：

1. 跟单成功/失败
2. 持仓变化
3. 净值刷新
4. 风险告警
5. 系统通知

## 2. 输入来源

1. `execution.result.v1`
2. `monitor.alert.v1`
3. `signal.equity.v1`
4. `position snapshot changed`

## 3. 推送模型

建议按用户和账户双维度订阅：

1. 用户维度：系统通知、全局告警
2. 账户维度：某个 MT5 账户的仓位、净值、执行状态

## 4. 连接管理

服务需要维护：

1. `session_id -> user_id`
2. `session_id -> subscribed_accounts`
3. `account_id -> session list`

这部分元数据建议放 Redis，便于横向扩容。

## 5. 背压与降级

盯盘场景下会出现高频推送，必须支持降级：

1. 高频净值更新可做合并推送
2. 非关键消息可采样
3. 单用户积压过多时，优先保留告警和执行结果

## 6. 第一阶段实现重点

先不要追求全量订阅能力，优先支持：

1. 单用户登录
2. 指定账户订阅
3. 跟单结果与基础告警推送
