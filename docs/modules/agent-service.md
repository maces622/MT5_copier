# Agent 调度服务

## 1. 职责

Agent Service 是未来扩展模块，用于接入 AI Agent，但必须被限制在平台权限体系内。

它的职责不是直接交易，而是：

1. 解析自然语言意图
2. 生成受控内部动作
3. 调用已有服务完成执行
4. 订阅事件并根据规则触发提醒或自动动作

## 2. 典型场景

1. “把所有跟单账户的黄金多头平掉”
2. “如果今天回撤超过 10%，暂停全部跟单”
3. “每天 22:00 推送账户净值摘要”

## 3. 执行原则

1. 大模型只做意图解析和参数结构化。
2. 真正执行仍通过 Account/Config、Copy Engine 或执行 Topic。
3. 高危动作必须满足权限校验，必要时要求人工确认。

## 4. 事件订阅

Agent 可以订阅：

1. `signal.equity.v1`
2. `execution.result.v1`
3. `monitor.alert.v1`

然后结合规则引擎形成条件动作。

## 5. 风险控制

1. Agent 不允许绕过风控服务。
2. Agent 产生的动作必须保留审计日志。
3. 自动执行策略必须支持启停和回滚。

## 6. 第一阶段状态

该模块先只保留设计，不进入第一阶段实施清单。等核心跟单闭环稳定后再接入。
